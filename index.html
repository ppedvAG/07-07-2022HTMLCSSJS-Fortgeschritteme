Emmet

Was ist Emmet?

Kurzform um HTML zu schreiben. Schneller und einfacher als alles selbst zu tippen. Früher bekannt als Zen-Coding.

Wie funktioniert es?

Ich schreibe den Elements-Namen und drücke tab.

Stellt auch weitere Funktionalität bereit wie das einsetzen von Id's und Klassen.

Emmet-Syntax

  • elementName + Tab => Erstellt öffnenden und (falls nötig) schließenden Tag
  • elementName.className + Tab => Erstellt Tag mit entsprechender Klasse
  • elementName#idName + Tab => Erstellt den Tag mit entsprechender Id
  • element1>element2 + Tab => Erstellt element2 als Child von element1
  • element1+element2 + Tab => Erstellt element1 und element2 als Siblings, also auf der selben Stufe
  • element1>element2^element3 + Tab => element 3 wird auf der selben Ebene wie element1 erstellt
  • element1*n + Tab => Erstellt element1 n-Mal
  • element1>(element2+element3)*4 + Tab => Würde viermal element2 benahcbart von element3 innerhalb von element1 erstellen
  • element1[attribut=wert] + Tab => Erstellt das Element mit dem entsprechenden Attribut
  • element1#$test + Tab => Stellt uns Nummerierung zur Verfügung, meistens in Kombi mit Multiplikator benutzt
  • element1#test$@n*3 + Tab => Erstellt element1 3 mal mit Nummerierung und n als Startzahl. Vor dem n kann noch ein Minus stehen, dann würde n die Endzahl sein. Steht das Minus alleine wird absteigend nummeriert
  • element1{text} + Tab => Erstellt element1 mit dem entsprechenden Text
  • Bei lorem kann ich direkt hinter lorem eine Zahl schreiben, also ohne das Mal-Zeichen, und dann werden mir entsprechend viele Wörter generiert

Article & Section

Können beliebig getauscht werden. Stehen beide für eigenständige Teile der Seite, die Informationen übermitteln. Sollten im Idealfall ein eigenes Heading haben. Können ineinander verschachtelt werden. Sind etwas besser für SEO als eine div, da sie ihre Funktion im Namen mitübermitteln. Die div hingegen gilt als leerer Tag. Gelten als inklusiver, da sie Screenreadern ihre Arbeite erleichtern.

Header & Footer

Können entweder als Kopf- und Fußzeile unsere gesamten Seite dienen, oder in Article und Section verwendet werden um zusätzliche Infos zu liefern.

Nav

Ist für die Navigationsleiste gedacht. Sollte nur die Navigationsleiste enthalten.

Aside Tag

Ist für interessante Infos innerhalb eines Textes gedacht, oder Bilder und weiterführende Infos.

Der picture-Tag

Der picture-Tag bietet uns eine alternative zum img-Tag. Der Tag ist responsive. Der Tag enhält mindestens einen source-Tag und genau einmal den img-Tag. Beim source-Tag gibt es noch das media-Attribut mit dem wir abfragen können wie groß der Bildschirm ist. Der img-Tag wird als Fallback benutzt, falls keiner der source-Tags passt oder der Browser den picture-Tag nicht unterstützt.

Der map-Tag

Der map-Tag bietet uns die Möglichkeit ein Bild in mehrere Bereiche einzuteilen und diese Bereiche zu verschiedenen Links zu leiten. Dabei müssen wir aufpassebn, dass wir die richtigen attribute verwenden, da das Bild selber auch noch das Attribut usemap="#nameDerMap" entahlten muss. Der map-Tag enthält mehrere area-Tags. Diese haben folgende Attribute:

  • shape-Attribut, kann folgende Werte haben:
    • rect, Viereckiger Bereich
    • circle, Runder Bereich
    • polygon, Vieleck
    • default, => selbe wie rect
  • coords, verhalten sich je nach shape unterschiedlich:
    • Rect: coords="x1, y1(obere linke Eck),x2,y2(rechte untere Eck)"
    • Circle: coords="x1,y1(Mittelpunkt des Kreises),r(Radius)"
    • Polygon: coords="x1,y1(erstes Ecke), x2,y2(zweites Eck)...."
  • href, Link wo bei klick hingeführt wird
  • alt, Text der angezeigt wird wenn ich über dem Bereich hovere
